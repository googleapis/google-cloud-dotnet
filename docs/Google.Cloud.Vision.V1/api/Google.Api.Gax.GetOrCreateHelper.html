Remarks

If fetcher fails with an RpcException with a status code of NotFound, it is assumed that the resource needs to be created, and creator is called. If this then fails with an RpcException with a status code of AlreadyExists, it is assumed that two clients have been racing to create the resource, and the whole process is retried. The maximum number of attempts (whole get/create cycles) is limited by maxAttempts.

If validator is not null, it is called when a resource is fetched rather than created, with the fetched resource being passed in. If the result of the validation is non-null, it is used as the message of a ResourceMismatchException which is thrown as the result of the operation.
